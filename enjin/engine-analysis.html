Key Design Principles

  • Physics abstracted behind IPhysicsBackend — swap Jolt/Box2D at runtime
  • Thread-safe ECS with deferred entity destruction and mutex-guarded structural ops
  • GPU-driven rendering: HiZ occlusion culling, clustered lighting, 64-bit material sort keys
  • Scene classification gates — 2D scenes skip shadow passes entirely

Codebase Health

  • ~653 source files across modular subsystems
  • 10+ security audits, 205+ findings resolved
  • Trust zones documented in .enjin-boundaries.json
  • Custom test framework: 80 CTest targets, 1100+ cases, all passing

03 — Rendering

Frame Pipeline

Each frame follows a classification-gated pipeline. 2D scenes skip shadow and RT passes entirely.

Scene Classify Determine Scene2D / Scene2_5D / Scene3D. Gates which passes run — 2D skips shadows, 2.5D skips shadows but keeps lighting.
Shadow Pass 4-cascade CSM (2048²) with cascade blending + Poisson PCF. Up to 4 point cubemaps (512²) and 4 spot maps (1024²). Caster list cached.
GPU Culling Two-phase HiZ occlusion culling via async compute. Builds Hi-Z pyramid from depth, then dispatches visibility tests. Feeds indirect draw buffer.
Cluster Assign 16×9×24 frustum grid. Each tile gets a light list. Reduces per-fragment lighting from O(N) to O(K) active lights.
Main Render Entities sorted by 64-bit material key (pipeline | material | texture). Descriptor cache tracks last-bound state — 75-85% cache hit rate. Opaque, then OIT, sprites, particles, vegetation, water.
Ray Tracing Optional. TLAS rebuild → dispatch RT shadows, reflections, AO, GI, translucency, caustics. Denoise via SVGF, Intel OIDN, or NVIDIA OptiX. Composite into final buffer.
Post-Processing 13-pass chain: bloom, SSAO, contact shadows, DoF (bokeh), tilt-shift, cel outlines, stipple/dither, color grading, TAA/FXAA, vignette, film grain, retro effects. Post-process volumes with spatial blending.
Present Editor overlay (ImGui) if in editor mode. Final swapchain present with fence-based GPU sync.

GPU-Side Optimizations

  • Two-phase HiZ occlusion culling (async compute)
  • Clustered forward lighting (16×9×24)
  • 64-bit material sort keys — pipeline+material+texture
  • Per-frame linear allocator — O(1) bump allocation
  • Descriptor set caching — m_LastBound tracking
  • LOD with hysteresis band (no popping)

CPU-Side Optimizations

  • Fence-based GPU sync (replaced vkDeviceWaitIdle)
  • Scene classification — 2D skips 4+ shadow passes
  • Spatial hash grid broad-phase — 4950→~200 pairs
  • Component flags — avoid 6+ optional GetComponent calls
  • Script query cache — 6 queries→1 per frame
  • Frame pacing — timeBeginPeriod(1) + 2ms spin
05 — Differentiators

What Sets TEGE Apart

Gameplay Systems Built In

Quest system, save/load (10 slots), dialogue trees, behavior tree AI with navmesh, HUD, destructibles, cinematics, object pooling, localization — all included, not plugins.

Accessibility Leadership

8 colorblind modes (GPU shader), screen reader with priority queue, switch access, dwell-click, dyslexia fonts, WCAG AAA themes, reduced motion, font scaling 0.5–3x.

Flash Game Revival

SWF binary import to ECS entities, AS2/AS3 transpiler, ~40 Flash API shim bindings, Newgrounds API (medals, scoreboards, cloud saves), Flash-style timeline editor.

Retro Art Pipeline

Built-in pixel editor with layers and 9 retro palettes. PSX flat shading, affine textures, vertex snapping, CRT scanlines, color quantization, stipple transparency.

7 Node Graph Editors

Shader graph (54 nodes, GLSL codegen), dialogue tree, behavior tree, quest flow, audio event graph, particle graph, and animation state machine — all with visual debugging.

Audited & Hardened

10+ security audits with 205+ findings resolved. Trust zones documented. Script sandbox with 1M instruction limit. Validated serializers. Published audit reports.
