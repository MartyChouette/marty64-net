TEGE

The Enjin Game Engine

A proprietary, licensable game engine built from scratch using C++20 and Vulkan API.

C++20 Vulkan 1.3 GLFW3 Dear ImGui ImGuizmo CMake

Features

Vulkan Renderer

Modern graphics pipeline with Blinn-Phong lighting and PBR materials. Depth buffer, SPIR-V shaders, swapchain management.

Entity-Component System

Flexible ECS architecture for game objects. Clean separation of data and behavior.

glTF Model Import

Load .gltf and .glb 3D models directly into the editor. Scene importer converts to ECS entities.

Editor UI

Full ImGui-based editor with hierarchy, inspector, viewport, and settings panels.

Transform Gizmos

Visual translate, rotate, and scale manipulation via ImGuizmo. Local and world space.

Entity Selection

Click-to-select with ray casting in the viewport.

PBR Materials

Base color, metallic, roughness, emissive. Alpha cutoff and transparency support.

Animation

2D sprite flipbook animation. 3D skeletal animation with bone hierarchy, skinning, blending, and state machines.

AI & Pathfinding

Enemy behaviors, FSM state machines, navmesh generation, A* pathfinding.

Procedural Generation

2D prefab-based levels with marked openings. 3D room/corridor systems.

Post-Processing

Bloom, tone mapping, vignette, color grading. Retro effects: PSX, CRT, dithering, vertex jitter.

Audio

Basic audio system with FMOD and Wwise integration hooks.

Architecture

Project Structure 
EnjinEngine/
+-- Core/           # Foundation layer (Memory, Math, Logging, Platform)
+-- Engine/         # Engine layer (Renderer, ECS, Assets, GUI)
+-- Editor/         # Editor application
+-- third_party/    # External dependencies (GLFW, ImGui, ImGuizmo)
+-- Tests/          # Unit tests

Core Layer

Foundation systems that everything else builds on: memory management (stack, pool, and linear allocators), math library (vectors, matrices, quaternions, matrix inversion), thread-safe logging, and platform abstraction.

Engine Layer

The Vulkan renderer, ECS, asset loading (glTF), input system, camera system, animation, AI, and audio all live here.

Editor Layer

The editor application built on top of the engine. ImGui for all UI, ImGuizmo for transform gizmos, and a fly camera for viewport navigation.

Vulkan Renderer

  • Vulkan context initialization and swapchain management
  • Command buffer system

  • PBR material system (baseColor, metallic, roughness, emissive)

Environmental

Entity-Component System

Flexible ECS architecture. Entities are IDs, components are pure data, systems operate on component groups. The glTF scene importer converts loaded models into ECS entities with the appropriate components (transform, mesh, material).

Animation

Roadmap

Phase 1: Foundation

Complete
  • Memory management (stack, pool, linear allocators)
  • Math library (vectors, matrices, quaternions, matrix inversion)
  • Logging system (thread-safe, categorized)
  • Platform abstraction layer
  • Entry point abstraction

Phase 2: Vulkan Renderer

Complete
  • Vulkan context initialization
  • Swapchain management
  • Command buffer system
  • SPIR-V shader pipeline
  • Depth buffer / Z-testing
  • Blinn-Phong lighting
  • Uniform buffer objects (MVP, lighting, material)

Phase 3: Engine Core

Complete
  • ECS (Entity Component System)
  • glTF asset loading (.gltf/.glb)
  • Scene importer (glTF to ECS conversion)
  • Input system (keyboard/mouse)
  • Camera system (fly camera with WASD + mouse)

Phase 4: Editor Tooling

Complete
  • Editor GUI (Dear ImGui integration)
  • Scene hierarchy panel
  • Entity inspector panel
  • Transform gizmos (ImGuizmo - translate/rotate/scale)
  • Entity selection via ray casting
  • Viewport panel with camera controls
  • Settings panel (gizmo options, render settings)
  • Stats overlay (FPS, frame time)

Phase 5: Advanced Rendering

In Progress

  • Multiple light sources (point, spot, directional)
  • Post-processing (bloom, tone mapping, vignette, color grading)
  • Retro effects (PSX, CRT, dithering, vertex jitter)
  • Weather system (rain, snow, fog)
  • Water rendering (waves, reflections, caustics)

Phase 6: Production Features

In Progress

Phase 7: Animation & Audio

Complete
  • 2D sprite animation (frame-based, flipbook)
  • 3D skeletal animation (bone hierarchy, skinning)
  • Animation blending & state machines
  • Audio system (basic)

Phase 8: AI & Procedural Generation

Complete
  • Spline system (linear, Bezier, Catmull-Rom, B-spline)

  • 2D procedural level generation (prefab-based with marked openings)

Building

Prerequisites

  • CMake 3.20+
  • C++20 compatible compiler (GCC 10+, Clang 12+, MSVC 2019+)
  • Vulkan SDK
  • GLFW3

Linux

Build commands 
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build . -j$(nproc)

Windows

Build commands 
mkdir build && cd build
cmake .. -G "Visual Studio 16 2019" -A x64
cmake --build . --config Release

Build Options

OptionDefaultDescription
ENJIN_BUILD_EDITORONBuild the editor
ENJIN_BUILD_TESTSOFFBuild unit tests
ENJIN_BUILD_EXAMPLESOFFBuild example projects
