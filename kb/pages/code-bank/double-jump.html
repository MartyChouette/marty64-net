VG101 / Code Bank / Double Jump

Double Jump

An extra jump in mid-air. Press jump again before landing to get additional height or change direction. The most common platformer upgrade.

Practice - what you do

The Feel

Double jump adds aerial control and recovery. It lets players correct mistakes, reach higher platforms, and feel more expressive in the air.

It's also a psychological safety net - players feel less committed to their first jump because they have a second chance.

Exposed Variables

Variable Type Default What it does
extraJumps int 1 Number of air jumps allowed
airJumpForce float 10.0 Force of air jump (often less than ground)
jumpsRemaining int - Current available air jumps
resetVelocity bool true Cancel downward velocity before air jump

Tuning Guide

For same-height double jump:

  • airJumpForce = same as jumpForce
  • resetVelocity = true
  • Each jump feels equal

For diminishing jumps:

  • airJumpForce = 70-80% of jumpForce
  • Second jump is weaker, encourages using first jump wisely

Reference: Hollow Knight, many Metroidvanias

For boosted air jump:

  • airJumpForce > jumpForce
  • resetVelocity = true
  • Air jump is the powerful one; first jump is setup

Pseudocode

On Landing:
  - Set jumpsRemaining = extraJumps

On Jump Press:
  - If grounded: do normal jump
  - Else if jumpsRemaining > 0:
    - If resetVelocity: set vertical velocity to 0
    - Apply airJumpForce upward
    - Decrement jumpsRemaining
    - Play air jump effect

Unity Setup (2D)

  1. Create Input Actions:
    • Window → Input Actions (or open existing asset)
    • Create "Jump" action (Button type) - bind to Space and A Button
    • Save and enable "Generate C# Class"
  2. Setup the Player:
    • Add Rigidbody2D (Gravity Scale: 3, Freeze Rotation Z: checked)
    • Add Collider2D (BoxCollider2D or CapsuleCollider2D)
    • Add PlayerInput component, assign your Input Actions asset
    • Add the DoubleJump2D.cs script below
  3. Create Ground Check:
    • Create empty child object named "GroundCheck"
    • Position it at the player's feet
    • Assign to the script's groundCheck field
  4. Configure Layers:
    • Edit → Project Settings → Tags and Layers
    • Create "Ground" layer, assign to ground objects
    • Set groundLayer in the script's Inspector
  5. Optional - Air Jump Effects:
    • Create a particle prefab for air jump (wings, sparkles)
    • Assign to airJumpEffect field
    • Assign different AudioClip for airJumpSound

The Script (2D)

// ============================================================
// DoubleJump2D.cs
// A 2D double jump with configurable air jumps, force options,
// and distinct feedback for ground vs air jumps.
// Unity 6.x + New Input System
// ============================================================

using UnityEngine;
using UnityEngine.InputSystem;

[RequireComponent(typeof(Rigidbody2D))]
public class DoubleJump2D : MonoBehaviour
{
    // ========================================================
    // JUMP PARAMETERS - Tune these in the Inspector
    // ========================================================

    [Header("Ground Jump")]
    [Tooltip("Force applied for ground jump")]
    [SerializeField] [Range(5f, 25f)] private float jumpForce = 12f;

    [Header("Air Jump")]
    [Tooltip("Number of air jumps allowed (1 = double jump, 2 = triple, etc.)")]
    [SerializeField] [Range(1, 5)] private int extraJumps = 1;

    [Tooltip("Force applied for air jump (often less than ground)")]
    [SerializeField] [Range(5f, 25f)] private float airJumpForce = 10f;

    [Tooltip("Cancel downward velocity before air jump? (Recommended: true)")]
    [SerializeField] private bool resetVelocity = true;

    [Header("Ground Detection")]
    [Tooltip("Empty transform at player's feet")]
    [SerializeField] private Transform groundCheck;

    [Tooltip("Radius of ground check circle")]
    [SerializeField] [Range(0.05f, 0.5f)] private float groundCheckRadius = 0.1f;

    [Tooltip("Which layers count as ground")]
    [SerializeField] private LayerMask groundLayer;

    [Header("Feedback - Air Jump")]
    [Tooltip("Particle effect spawned on air jump (optional)")]
    [SerializeField] private GameObject airJumpEffect;

    [Tooltip("Sound for air jump (should differ from ground jump)")]
    [SerializeField] private AudioClip airJumpSound;

    [Tooltip("Sound for ground jump")]
    [SerializeField] private AudioClip groundJumpSound;

    [Header("Input Actions")]
    [Tooltip("Reference to your Input Actions asset")]
    [SerializeField] private InputActionAsset inputActions;

    [Header("Debug")]
    [Tooltip("Show debug info in console")]
    [SerializeField] private bool debugMode = false;

    // ========================================================
    // INTERNAL STATE - Don't modify directly
    // ========================================================

    private Rigidbody2D rb;
    private AudioSource audioSource;
    private InputAction jumpAction;

    // Jump state
    private bool isGrounded = false;
    private bool wasGrounded = false;
    private int jumpsRemaining = 0;

    // ========================================================
    // PUBLIC PROPERTIES - For other scripts to read
    // ========================================================

    /// <summary>
    /// True if player is on the ground
    /// </summary>
    public bool IsGrounded => isGrounded;

    /// <summary>
    /// Current remaining air jumps
    /// </summary>
    public int JumpsRemaining => jumpsRemaining;

    /// <summary>
    /// Maximum air jumps (same as extraJumps)
    /// </summary>
    public int MaxAirJumps => extraJumps;

    /// <summary>
    /// True if player can currently jump (grounded or has air jumps)
    /// </summary>
    public bool CanJump => isGrounded || jumpsRemaining > 0;

    // ========================================================
    // UNITY LIFECYCLE
    // ========================================================

    void Awake()
    {
        // Get required components
        rb = GetComponent<Rigidbody2D>();
        audioSource = GetComponent<AudioSource>();

        // Initialize jumps
        jumpsRemaining = extraJumps;

        // Setup input actions
        if (inputActions != null)
        {
            var gameplay = inputActions.FindActionMap("Gameplay");
            if (gameplay != null)
            {
                jumpAction = gameplay.FindAction("Jump");
            }
        }
    }

    void OnEnable()
    {
        if (jumpAction != null)
        {
            jumpAction.Enable();
            jumpAction.performed += OnJumpPressed;
        }
    }

    void OnDisable()
    {
        if (jumpAction != null)
        {
            jumpAction.performed -= OnJumpPressed;
            jumpAction.Disable();
        }
    }

    void Update()
    {
        // Check ground state
        CheckGroundState();
    }

    // ========================================================
    // INPUT HANDLING
    // ========================================================

    /// <summary>
    /// Called when jump input is pressed
    /// </summary>
    private void OnJumpPressed(InputAction.CallbackContext context)
    {
        TryJump();
    }

    /// <summary>
    /// Attempt to perform a jump (ground or air)
    /// </summary>
    public void TryJump()
    {
        if (isGrounded)
        {
            // Ground jump
            PerformGroundJump();
        }
        else if (jumpsRemaining > 0)
        {
            // Air jump (double jump)
            PerformAirJump();
        }
        else
        {
            if (debugMode) Debug.Log("Jump blocked: not grounded, no air jumps");
        }
    }

    // ========================================================
    // JUMP EXECUTION
    // ========================================================

    /// <summary>
    /// Execute a ground jump
    /// </summary>
    private void PerformGroundJump()
    {
        // Apply jump force
        rb.linearVelocity = new Vector2(rb.linearVelocity.x, jumpForce);

        // Play ground jump sound
        PlaySound(groundJumpSound);

        if (debugMode) Debug.Log("Ground jump!");
    }

    /// <summary>
    /// Execute an air jump (double jump)
    /// </summary>
    private void PerformAirJump()
    {
        // Optionally reset vertical velocity first
        // This ensures consistent air jump height regardless of fall speed
        if (resetVelocity)
        {
            rb.linearVelocity = new Vector2(rb.linearVelocity.x, 0f);
        }

        // Apply air jump force
        rb.linearVelocity = new Vector2(rb.linearVelocity.x, airJumpForce);

        // Consume one air jump
        jumpsRemaining--;

        // Spawn air jump effect
        SpawnAirJumpEffect();

        // Play air jump sound (different from ground jump!)
        PlaySound(airJumpSound);

        if (debugMode)
        {
            Debug.Log($"Air jump! Jumps remaining: {jumpsRemaining}");
        }
    }

    // ========================================================
    // GROUND DETECTION + JUMP REFRESH
    // ========================================================

    /// <summary>
    /// Check ground state and refresh air jumps when landing
    /// </summary>
    private void CheckGroundState()
    {
        wasGrounded = isGrounded;

        // Check for ground
        if (groundCheck != null)
        {
            isGrounded = Physics2D.OverlapCircle(
                groundCheck.position,
                groundCheckRadius,
                groundLayer
            );
        }

        // Just landed - refresh air jumps!
        if (isGrounded && !wasGrounded)
        {
            jumpsRemaining = extraJumps;

            if (debugMode)
            {
                Debug.Log($"Landed! Air jumps refreshed to {extraJumps}");
            }
        }
    }

    // ========================================================
    // FEEDBACK
    // ========================================================

    /// <summary>
    /// Spawn visual effect for air jump
    /// </summary>
    private void SpawnAirJumpEffect()
    {
        if (airJumpEffect != null)
        {
            // Spawn at player position
            GameObject effect = Instantiate(
                airJumpEffect,
                transform.position,
                Quaternion.identity
            );

            // Auto-destroy after 2 seconds
            Destroy(effect, 2f);
        }
    }

    /// <summary>
    /// Play a sound clip
    /// </summary>
    private void PlaySound(AudioClip clip)
    {
        if (clip != null && audioSource != null)
        {
            audioSource.PlayOneShot(clip);
        }
    }

    // ========================================================
    // PUBLIC METHODS - Call from other scripts
    // ========================================================

    /// <summary>
    /// Manually refresh air jumps (e.g., touching wall, taking damage)
    /// </summary>
    public void RefreshAirJumps()
    {
        jumpsRemaining = extraJumps;
    }

    /// <summary>
    /// Grant bonus air jumps (e.g., power-up)
    /// </summary>
    public void AddAirJump(int amount = 1)
    {
        jumpsRemaining = Mathf.Min(jumpsRemaining + amount, extraJumps);
    }

    // ========================================================
    // DEBUG VISUALIZATION
    // ========================================================

    void OnDrawGizmosSelected()
    {
        // Show ground check radius
        if (groundCheck != null)
        {
            Gizmos.color = Color.green;
            Gizmos.DrawWireSphere(groundCheck.position, groundCheckRadius);
        }
    }
}

// ============================================================
// USAGE NOTES:
//
// 1. The key difference from BasicJump: we track jumpsRemaining
//    which decrements on air jump and resets on landing.
//
// 2. Air jump MUST have different feedback than ground jump!
//    Players need to know they used their resource. Use:
//    - Different sound (airJumpSound vs groundJumpSound)
//    - Visual effect (airJumpEffect - wings, sparkles, etc.)
//    - Different animation (not shown, but recommended)
//
// 3. resetVelocity = true is recommended. Without it, air jump
//    while falling fast gives much less height than air jump
//    at apex of first jump. With it, height is consistent.
//
// 4. To integrate with coyote time: coyote jump should NOT
//    consume an air jump. Check "just left ground" separately
//    from "in air with jumpsRemaining".
//
// 5. For wall touch reset, call RefreshAirJumps() on wall contact:
//    void OnCollisionEnter2D(Collision2D col)
//    {
//        if (col.gameObject.layer == LayerMask.NameToLayer("Wall"))
//            GetComponent<DoubleJump2D>().RefreshAirJumps();
//    }
//
// 6. For 3D: Change Rigidbody2D to Rigidbody, Physics2D to
//    Physics.CheckSphere, and use rb.linearVelocity with Vector3
// ============================================================

Common Issues

"Double jump doesn't feel different from first jump"
Add distinct feedback: different sound, particle effect, animation. Players should know they used their air jump.

"It feels like I have infinite jumps"
Make sure you're only resetting jumpsRemaining on landing, not on jump. Also check that the counter actually decrements.

"Air jump barely helps when falling fast"
If resetVelocity is false, the jump force fights existing downward velocity. Set it to true to always get full air jump height.

"Players are reaching places they shouldn't"
Double jump significantly increases vertical reach. Redesign level geometry or reduce airJumpForce.

Combine With

  • Variable Jump - both jumps can be variable height
  • Coyote Time - coyote jump shouldn't consume air jump
  • Dash - dash + double jump for maximum air mobility
  • Squash & Stretch - unique deformation for air jump

Design Decisions

Question Options
When to unlock double jump? Start with it, ability pickup, permanent upgrade
Should coyote time use an air jump? Most videogames: no. Coyote is a grace period, not a true air jump.
Reset air jump on wall touch? Yes for wall-jump videogames, creates infinite climbing
Reset air jump on taking damage? Generous design choice, prevents unfair deaths
Craft - for educators

Why Teach This

Double jump teaches ability management. Students learn to track state (jumps remaining), reset conditions (landing), and provide feedback (air jump effects). It's more complex than basic jump but still approachable.

Teaching Sequence

  1. Start with basic jump working
  2. Add counter for remaining jumps
  3. Allow jump when jumpsRemaining > 0 and in air
  4. Add distinct visual/audio feedback for air jump
  5. Tune the feel (force, velocity reset)

The Feedback Lesson

Double jump is a great place to teach feedback differentiation. The air jump needs to feel different from the ground jump so players know they've used their resource. This could be:

  • Different sound effect
  • Different particle effect (wings, sparkles)
  • Different animation
  • Brief screen flash or shake

Without distinct feedback, players lose track of their jumps.

Assessment Questions

  • Why might you want the air jump to be weaker than the ground jump?
  • What feedback tells the player they've used their double jump?
  • How does double jump change level design requirements?
Theory - deeper grounding

Heritage Notes

The double jump has murky origins but became codified in the 1980s-90s. Dragon Buster (1984) is often cited as an early example. The technique spread through action-platformers and Metroidvanias.

It's now so common that players expect it. Many videogames start without double jump and grant it as an upgrade - following the Metroidvania tradition of ability-gated progression.

The Abstraction Question

Double jump is physically impossible. You can't push off nothing. This is fine - videogames aren't physics simulations. But it raises the question: how much do videogame physics need to make sense?

The answer is usually: they need to be internally consistent, not realistic. If double jump exists, it should work the same way every time. Players learn videogame physics, not real physics.

Permission and Progression

From a Permissions perspective, double jump expands what's allowed. Before the upgrade, certain heights are forbidden. After, they're reachable. This is the Metroidvania formula: new abilities = new permissions = new areas.

Triple Jump and Beyond

Why stop at two? Some videogames allow triple jumps or more. But there's diminishing returns:

  • More jumps = more to track mentally
  • More jumps = harder to design levels for
  • More jumps = less meaningful each jump becomes

Double jump is the sweet spot for most videogames. It adds expression without overwhelming.
